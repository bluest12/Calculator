⚠️ 注意事项 / Notes

1. 关于运行环境

这个计算器扩展是专门为有自定义扩展做的，别的平台我没试过，应该用不了

如果你发现导入后积木显示不出来，大概率是平台不对，或者是文件出问题

2. 关于分数计算

分数输入要用"/"，比如“1/2+1/3”这样写，书上大概是一条线

分数加分数、分数乘分数这些都没问题，结果会保持分数形式

但是算式里出现小数，结果就会变成小数，比如“1/2+0.5”结果是“1”，不是“2/2”

这个设计可能后续版本会修复

3. 关于%运算符

“%”现在只能算整数、小数取余，比如“10%3”结果是“1”

别拿它算“1/2%0.3”，会显示“NaN”

4. 关于比较运算符

“>”、“<”、“=”、“≤”、“≥”这几个都能比较分数和小数，放心用，不会出现“NaN”这样的文本

“≡”是全等比较，分数和小数即使数值相等也会返回false，比如“1/2≡0.5”就是false

比较结果返回的是布尔值，也就是true或者false

1. Regarding the operating environment 

This calculator extension is specifically designed for custom extensions. I haven't tried it on other platforms, so it probably won't work. 

If you find that the blocks don't show up after importing, it's most likely due to an incorrect platform or a problem with the file. 

2. Regarding Score Calculation 

Fractions should be input using "/", for example, write "1/2 + 1/3" like this. In the book, it's approximately a horizontal line. 

Adding fractions to fractions and multiplying fractions by fractions are all fine. The results will remain in fractional form. 

However, if there are decimals in the equation, the result will become a decimal. For example, the result of "1/2 + 0.5" is "1", not "2/2". 

This design issue might be fixed in a later version. 

3. About the % Operator 

The "%" symbol currently only calculates the remainder of integers and decimals. For example, the result of "10%3" is "1". 

Don't calculate it as "1/2%0.3", or it will show "NaN". 

4. About Comparison Operators 

The symbols “>”, “<”, “=”, “≤”, and “≥” can all be used to compare fractions and decimals. Feel free to use them; there won't be any text like "NaN". 

"≡" represents an exact equality comparison. Even if the values of fractions and decimals are the same, it will return false. For example, "1/2 ≡ 0.5" is false. 

The comparison result returns a Boolean value, that is, true or false.
